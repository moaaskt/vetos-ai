## Método de Assinatura do Tutor

| Option | Description | Selected |
|--------|-------------|----------|
| Assinatura desenhada em tela | Desenho da assinatura via canvas (touchscreen ou mouse) | |
| Aceite eletrônico estruturado | Checkbox de aceite com dados de Nome, CPF, IP, data/hora e User-Agent | ✓ |

**User's choice:** Aceite eletrônico estruturado (Nome + CPF + IP + Data/Hora + User-Agent).
**Notes:** O tutor deve assinar através de uma página pública acessada pelo link compartilhado pela clínica. Não é necessário desenho de tela nesta fase, pois o aceite com evidências coletadas é suficiente.

## Ciclo de Vida e Estados do Documento

| Option | Description | Selected |
|--------|-------------|----------|
| Adição de flag ao ConsentTerm | Adicionar campos de auditoria e a flag `tutorSigned` ao modelo existente | ✓ |
| Criação de novo modelo ou status global | Criar uma tabela de auditoria ou novo status no enum principal | |

**User's choice:** Adição de campos de auditoria diretamente no modelo `ConsentTerm`.
**Notes:** O documento mantém seu status original (`SIGNED`) e o hash/QR Code válidos, mas registra na flag `tutorSigned = true` e nos novos campos de auditoria que a validação foi realizada. O backend rejeita assinaturas se o termo estiver em `DRAFT`.
