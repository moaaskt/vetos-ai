## Gatilho de Envio

### Q1: Onde deve ficar o botão "Enviar ao Tutor" na UI?

| Option | Description | Selected |
|--------|-------------|----------|
| Botão no modal de preview | Após assinatura, botão aparece no modal fullscreen ao lado de "Imprimir" | |
| Botão no dropdown de ações | Dropdown do PetDetails.tsx ganha opção "Enviar ao Tutor" para documentos SIGNED | |
| Ambos | Botão no modal de preview + opção no dropdown de ações da timeline para reenvio | ✓ |

**User's choice:** Ambos — botão no modal de preview para fluxo imediato pós-assinatura + opção no dropdown da timeline para reenvios futuros.
**Notes:** Motivação: melhor experiência para envio imediato, permite reenvio rápido de documentos antigos, mantém o prontuário como ponto central de consulta e compartilhamento.

## Rastreabilidade do Compartilhamento

### Q1: Como registrar e exibir o histórico de compartilhamentos?

| Option | Description | Selected |
|--------|-------------|----------|
| Novo event type no NotificationLog | Adicionar DOCUMENT_SHARED ao NotificationLog existente, com campo documentId opcional | |
| Campos sharedAt + sharedVia nos modelos | Adicionar campos diretamente em Prescription/ConsentTerm, sem NotificationLog | |
| Híbrido | NotificationLog para histórico detalhado + lastSharedAt nos modelos para acesso rápido | ✓ |

**User's choice:** Abordagem híbrida. NotificationLog para auditoria completa + lastSharedAt nos modelos.
**Notes:** NotificationLog preserva auditoria por envio (canal, destinatário, sucesso/falha, reenvios). Campo lastSharedAt facilita exibição rápida na timeline sem joins pesados. Events: PRESCRIPTION_SHARED e CONSENT_TERM_SHARED.
