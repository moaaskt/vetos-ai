## Decisões Estruturais de Modelagem (Database Design)

### 1. Modelo `Client` Expandido no Prisma
Para suportar os novos requisitos cadastrais e garantir compatibilidade retroativa, adicionaremos os seguintes campos ao modelo `Client`:

*   **Identificação:**
    *   `cpf` (String?, Nullable): Documento de identificação. Para suporte multi-tenant com CPF nulo, utilizaremos uma restrição de unicidade composta `@unique([clinicId, cpf])` se suportado pelo Prisma com valores nulos, ou trataremos a unicidade lógica no nível da aplicação/banco de forma segura.
    *   `rg` (String?, Nullable): Registro geral opcional.
    *   `birthDate` (DateTime?, Nullable): Data de nascimento.
*   **Contatos Expandidos:**
    *   `whatsapp` (String?, Nullable): Canal principal de mensageria automática.
    *   `emailAlt` (String?, Nullable): E-mail secundário.
    *   `emergencyName` (String?, Nullable): Nome do contato de emergência.
    *   `emergencyPhone` (String?, Nullable): Telefone do contato de emergência.
*   **Endereço Estruturado:**
    *   `postalCode` (String?, Nullable): CEP.
    *   `street` (String?, Nullable): Logradouro.
    *   `number` (String?, Nullable): Número residencial/comercial.
    *   `complement` (String?, Nullable): Complemento do endereço.
    *   `neighborhood` (String?, Nullable): Bairro.
    *   `city` (String?, Nullable): Cidade.
    *   `state` (String?, Nullable): Estado (UF, 2 letras).

```prisma
// Exemplo conceitual do modelo Client atualizado:
model Client {
  id               String            @id @default(uuid())
  name             String
  email            String?
  phone            String?
  clinicId         String
  createdAt        DateTime          @default(now())
  updatedAt        DateTime          @updatedAt
  clinic           Clinic            @relation(fields: [clinicId], references: [id])
  pets             Pet[]
  appointments     Appointment[]
  notificationLogs NotificationLog[]

  // Novos campos estruturados
  cpf              String?
  rg               String?
  birthDate        DateTime?
  whatsapp         String?
  emailAlt         String?
  emergencyName    String?
  emergencyPhone   String?
  postalCode       String?
  street           String?
  number           String?
  complement       String?
  neighborhood     String?
  city             String?
  state            String?

  @@unique([clinicId, cpf]) // Unicidade composta por tenant para CPFs preenchidos
}
```

### 2. Tratamento Técnico da Unicidade de CPF com Valores Nulos
No PostgreSQL, o Prisma mapeia `@@unique([clinicId, cpf])` de forma que múltiplos registros com o mesmo `clinicId` e `cpf = NULL` possam coexistir sem violar a restrição de unicidade (uma vez que `NULL != NULL` no padrão SQL). Isso garante a compatibilidade retroativa perfeita para clientes existentes sem CPF.
